# SAM による髭検出アプリケーション（白髭・太さ別の検出に対応）

SAM (Segment Anything Model) を使用した髭検出アプリケーション

## 概要

SAMのAutomatic Mask Generation (AMG) を活用して、画像内の髭を高精度に検出するGradioベースのWebアプリケーションです。

**新機能:**
- **毛質別のカウント**: 髭を太さ（幅）に基づいて4種類に分類（剛毛・硬毛・中間毛・軟毛）
- **色別のカウント**: 黒髭（肌より暗い）・白髭（肌より明るい）を個別にカウント
- **モード切替**: 2つの検出モードをUIで簡単に切り替え可能
- **自由な範囲指定**: フリーハンドで任意形状の領域を選択可能

## 必要環境

### Python パッケージ

```bash
pip install -r requirements.txt
```

> 古いバージョン（4.x系など）では `ImageEditor` コンポーネントの API 情報生成でエラーが発生する場合があります。

## 使い方

### 基本的な流れ

1. **画像をアップロード** - Input Image エリアに画像をドロップ
2. **検出モードを選択** - 「毛質別のカウント」または「色別のカウント」
3. **検出領域を指定** - 自由な範囲指定・矩形描画・座標入力から選択
4. **Hair Color Class を選択** - `黒ヒゲ`（黒髭）または `白ヒゲ`（白髭）
5. **検出開始 をクリック** - 検出実行

### 検出モード

#### 毛質別のカウント
髭の太さ（外接矩形の短辺）で4種類に分類:

| カテゴリ | 色 | 説明 |
|---------|---|------|
| 剛毛 (Coarse) | 赤 | 非常に太い髭 |
| 硬毛 (Thick) | オレンジ | 太い髭 |
| 中間毛 (Medium) | 緑 | 中程度の髭 |
| 軟毛 (Fine) | 紫 | 細い髭 |

太さ閾値をスライダーで調整可能。検出後、「幅の統計」を参考に閾値を微調整できます。

#### 色別のカウント
髭を明るさで2種類に分類:

| クラス | 説明 | 用途 |
|--------|------|------|
| **黒ヒゲ** | 肌より暗い髭 | 濃い髭、黒い髭の検出 |
| **白ヒゲ** | 肌より明るい髭 | 白髪の髭、グレーの髭の検出 |

各髭は虹色のレインボーカラーで個別に色分け表示されます。

### 領域指定方法

#### 方法1: 自由な範囲指定（フリーハンド）
1. Selection Mode で `自由な範囲指定` を選択
2. 画像上でペン（ブラシ）で囲みたい領域を線で囲む
3. 閉じた領域の内側が自動検出される

#### 方法2: 矩形描画
1. Selection Mode で `矩形描画` を選択
2. 画像上で矩形をドラッグして描画

#### 方法3: 座標入力
1. Selection Mode で `座標` を選択
2. X1, Y1（左上）と X2, Y2（右下）を入力
3. "Preview Region" で確認

## パラメータ説明

### SAM Settings（共通）

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| SAM Points Per Side | 64 | サンプリング密度。高いほど多くのマスク生成（64-96推奨） |

### 毛質別のカウント用パラメータ

#### Filter Parameters（フィルタ）

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| 最小面積 | 5 | 最小マスク面積（ピクセル） |
| 最大面積 | 100 | 最大マスク面積（ピクセル） |
| 最小アスペクト比 | 1.0 | アスペクト比フィルタ（細長さ） |
| 明るさの閾値 | 1.00 | 黒髭: マスク < 平均×閾値、白髭: マスク > 平均×閾値 |
| Dilation Kernel Size | 0 | 膨張処理カーネルサイズ（0=OFF、奇数推奨） |
| Dilation Iterations | 1 | 膨張処理の繰り返し回数 |

#### 太さ閾値

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| 剛毛閾値 | 8.0 px | この値以上が剛毛 |
| 硬毛閾値 | 6.0 px | この値以上が硬毛（剛毛未満） |
| 中間毛閾値 | 4.0 px | この値以上が中間毛（硬毛未満） |

### 色別のカウント用パラメータ

#### ヒゲの検出用パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| 最小面積 | 5 | 最小マスク面積（ピクセル） |
| 最大面積 | 2000 | 最大マスク面積（ピクセル） |
| 最小アスペクト比 | 1.2 | アスペクト比フィルタ（細長さ） |
| 明るさの閾値 | 1.14 | 黒髭: マスク < 平均×閾値、白髭: マスク > 平均×閾値 |
| Dilation Kernel Size | 3 | 膨張処理カーネルサイズ（0=OFF、奇数推奨） |
| 膨張処理の繰り返し回数 | 1 | 膨張処理の繰り返し回数 |

#### Brightness Threshold の仕組み

**検出の核となるパラメータ:**

| クラス | 判定ロジック |
|--------|------------|
| 黒髭 | マスク明るさ < ROI平均×閾値 → 検出 |
| 白髭 | マスク明るさ > ROI平均×閾値 → 検出 |

- **黒髭**: 閾値を上げる（1.2-1.3）→ より明るい髭も許容（検出数増加）
- **白髭**: 閾値を下げる（0.85-0.90）→ より暗い髭も許容（検出数増加）

### 検出したマスクの重複除去

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| Overlap Threshold | 0.5 | 重複判定の閾値。高い値=多く保持、低い値=厳しく除去 |

### Tile Processing（大きい画像用）

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| Enable Tile Processing | OFF | タイル分割処理を有効化 |
| Tile Size | 400 | 各タイルのサイズ（px） |
| Tile Overlap | 50 | タイル間のオーバーラップ（px） |

### 検出結果の可視化

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| Overlay Alpha | 0.4 | オーバーレイ透明度（0.2-0.5推奨） |
| Show Center Markers | ON | 髭の中心にマーカー表示 |

## 出力

| 出力 | 説明 |
|------|------|
| 検出結果（色 = 毛質） | 検出結果のオーバーレイ画像 |
| 毛質別のマスク | 毛質別モード: カテゴリ別に色分けされたマスク |

## チューニングのヒント

### 毛質別のカウントモード

#### 太さ分類がうまくいかない場合
1. まず検出を実行
2. ステータスの「幅の統計」でMin/Max/Avgを確認
3. 太さ閾値を調整（剛毛 > 硬毛 > 中間毛 の順）

### 色別のカウントモード

#### 黒髭が検出されない場合
- `明るさの閾値` を上げる (1.2-1.3)
- `最小面積` を下げる (3-5)

#### 白髭が検出されない場合
- `明るさの閾値` を下げる (0.85-0.90)
- `最小面積` を下げる (3-5)

#### 誤検出が多い場合
- `最小アスペクト比` を上げる (1.5-2.0)
- 黒髭: `明るさの閾値` を下げる
- 白髭: `明るさの閾値` を上げる

### 共通

#### 細い髭が検出されない場合
- `SAM Points Per Side` を 80-96 に上げる
- `最小アスペクト比` を 1.1-1.2 に下げる

#### 処理が遅い場合
- `SAM Points Per Side` を 48-56 に下げる
- GPU（CUDA/MPS）を使用する
- 検出領域を小さくする
